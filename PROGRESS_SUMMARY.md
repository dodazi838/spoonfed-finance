# 진행 상황 요약 (Progress Summary)

## 주요 수정 사항
1. **API 오류 처리 강화 (429 & 503)**
   - Gemini API의 Rate Limit (429 Too Many Requests) 오류 및 서버 지연 (503 Service Unavailable) 오류를 감지하고, 프론트엔드에서 사용자에게 친절한 안내 메시지를 띄우도록 수정했습니다.
   - pi/analyze/route.ts 및 pi/analyze-chapter/route.ts에 예외 처리 로직 추가.

2. **목차 추출 프롬프트 대폭 개선**
   - 명시적인 목차(Table of Contents) 페이지가 없는 PDF 문서에서도 AI가 본문의 글씨 크기나 굵기를 기반으로 핵심 대제목(Section Headers)을 스캔하여 스스로 목차를 구성하도록 프롬프트를 고도화했습니다.

3. **토큰 사용량 표시 및 모델 선택 기능 추가**
   - UI에 Flash (권장) 모델과 Lite (2.5) 모델을 선택할 수 있는 토글 버튼을 추가했습니다.
   - 리포트 결과 화면 상단에 현재 세션에서 소모한 전체 토큰 사용량을 시각적으로 표시하도록 ReportResult.tsx를 개선했습니다.

4. **UI 테마 및 텍스트 가독성 개선**
   - 웹앱 상단의 회색 텍스트가 라이트 모드(하얀 배경)에서도 잘 보이도록 명도 대비(Slate 계열 색상)를 조정하여 디자인 퀄리티와 가독성을 높였습니다.
   - 해설 텍스트가 한 덩어리로 뭉치지 않도록 문단을 나누어 출력하게끔 프롬프트 규칙을 추가했습니다.

5. **빌드 에러(SWC Parsing Error) 완벽 해결**
   - 템플릿 리터럴 내의 마크다운 기호(백틱)로 인해 발생하던 Expected a semicolon 등의 빌드 오류를 원천 차단하기 위해, 백틱 기호 없이 순수 JSON 응답만 주고받도록 시스템을 안정화했습니다.
